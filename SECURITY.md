# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OpenMemo, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email the project maintainers directly.

We will acknowledge your report within 48 hours and provide a timeline for a fix.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Security Considerations

OpenMemo stores and processes memory data. When using OpenMemo:

- Store sensitive data encrypted at rest
- Use environment variables for API keys
- Review storage adapter security for your deployment
- Keep dependencies updated
